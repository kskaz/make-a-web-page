Introduction to Programming Notes

Lesson 1: Basics of the Web

The Web

The web is a collection of documents which are made out of HTML. HTML stands for Hypertext Markup Language. HTML glues everything (text, videos, images, etc) together. Documents can contain links to other HTML documents. This interconnection of documents gives the Web its name.

Major Components of the Web

Your computer uses an application called a browser to view HTML documents that reside on other servers on the Internet. It uses the HTTP protocol to send and receive this information.

Composition of HTML

HTML is made up of:

  • text content (what you see)
  • markup (determines what it looks like)
  • references to other documents (images, videos, etc.)
  • links to other HTML pages

HTML uses tags to change the appearance of content in a document. They start with an opening tag and end with the closing tag. The closing tag shares the same name as the opening tag but also has a slash in front of the tag's name. The content being modified by the tag lies between the opening and closing tag.

Some tags we learned about:

  • bold
  • italics

HTML attributes

HTML attributes are tags that include attributes within them. These attributes have values.

Examples are:

  • anchor tag - links to another page
  • image tag - displays an image on the page

Some tags do not have closing tags if there is no content - the image tag is one of these.

Whitespace

In HTML, all whitespace turns into a single space. We use tags to control how whitespace appears on our page.

Example tags:

  • br - line break (void tag)
  • p - paragraph - has opening and closing tag - content between tags will be contained in the paragraph

Block vs Inline

Inline tags are just text. Block tags draw invisible boxes around its content and can have a height and width.

HTML structure

A complete HTML document includes:

  • doctype: type of document
  • html tag: at beginning and end of document
  • head tag: contains title tag and other meta data (CSS and JavaScript)
  • body tag: contains document content

We are human but computers are not...

Computers can only do what we ask them to do. If we make a mistake in our programming, unexpected results may occur. If we forget how to do something, we know we can always look it up. It's impossible to remember everything.

Lesson 2: Webpages, Documents and Structure

House Analogy

A website can be compared to a house. The HTML documents are like the rooms in a house. They define what and where the rooms are. CSS is the look and feel of a webpage. In our house comparision, this would be the color of the walls, the type of carpeting, and the decorations on the walls. Javascript allows interactivity on the webpage, much like a garage door opener and a TV remote allow us to interact with objects in our house.

Useful Tools for Creating Websites

Google Chrome offers a useful developer tool within the browser which allows you to view the structure of a page as the browser interprets it. You can also use it to modify values in the page to see how they affect the look of the page. The tool will also let you visually see the tree-like structure of how the browser interprets the webpage. There are also several editors you can use to create websites, such as Scratchpad.io, CodePen.io, and Sublime Text.

HTML, CSS and DOM

HTML is the language of the web - its syntax and rules. The basic word of this language is the tag. The browser turns these tags into elements of a tree structure (DOM). HTML defines the structure and content.

CSS defines the style (look and feel) of the webpage. It has its own syntax and rules.

DOM stands for Document Object Model. It is an independent convention for representing and interacting with objects in HTML and other markup languages. The nodes of every document are organized in a tree structure called the DOM tree.

Webpages are Composed of Boxes

Webpages can be broken up into several boxes - even boxes within boxes. Breaking up a webpage in this way makes it much easier to determine how each part of the webpage should be coded. Start with the big boxes first before worrying about the smaller boxes.

Div tags are used to define boxes. To apply style, you can mark these div sections by giving each element a class attribute to define what the box is.

Lesson 3: Adding CSS Style to HTML Structure

Don't Repeat Yourself

You want to avoid repetition as much as possible in your code. This reduces the amount of code you need to have in your webpage and allows you to easily make changes. Webpages will also load faster. You can use CSS to avoid repetition.

Cascading Style Sheets

CSS stands for "Cascading Style Sheets." Each sheet is a different layer of style, and you can have multiple sheets (files) in your code. Cascading means that the most specific rules for each element will be applied. An element's descendents in the HTML tree structure can inherit any text-related properties, but not box-related properties.

Applying Style

In your CSS file, you use selectors to specify how your elements look. You can apply styles to any tag, or you can create classes. The attributes and their values are enclosed in brackets for each element. Use the CSS documentation when needed. You can also put your CSS style code in the HTML document if the project is small. Putting style inline with your HTML is not recommended.

Box Revisited

The box model contains areas for content, padding, border, and margin. This box and its areas can be viewed using the developer tools in Google Chrome. The size of your element is equal to the sum of the border, padding and content. Don't forget to include the border and padding on both sides of the content. You can use the attribute "box-sizing" set to the value "border-box" to have the border and padding automatically calculated in the element size. Margin is not included in this size. Since this is a recent feature, you will need to add browser specific prefixes to ensure older browsers will display the content correctly.

You can also use percentages instead of pixels to define the size of your boxes. This will allow you to auto-adjust the size of your boxes for various browser screen sizes. The attribute maxwidth can be used to ensure your box never goes beyond a certain size, but it will also auto-adjust for smaller sizes as needed.

Positioning Boxes

Boxes by default take up the entire width of a page. To have boxes appear next to each other, you can use the flexbox layout. Use the attribute display with the value flex within the parent selector's brackets. The size of the elements must be smaller than the default 100%.

Code, Test and Refine

Repeat the process of modifying code and testing until the website looks correct.

  1. Look for natural boxes
  2. Look for repeated styles and semantic elements
  3. Write your HTML
  4. Apply styles (from biggest to smallest)
  5. Fix things

Use Google Chrome's developer tools to test out changes without having to alter your HTML code.

Verifying Your Code

Use the W3C validator websites to verify both your HTML and CSS code. This will ensure your website complies with the standard.
